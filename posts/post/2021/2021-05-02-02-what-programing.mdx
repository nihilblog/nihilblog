이 블로그는 프로그래밍과 프로그래밍 언어에 대해서 우선적으로 다룰 생각입니다. 하지만 곧바로 프로그래밍 언어에 대해 다루기 이전에, **프로그래밍이란 무엇인가**에 대해서 먼저 접근하는 것이 좋은 것이 아닐까? 라는 생각이 들었습니다. 이번 글에서는 그것에 대해서 이야기를 해보려고 합니다.

# 프로그래밍?
여러분은 프로그래밍에 대해 어떻게 생각하십니까? 프로그래밍은 무엇일까요?

답은 여러가지일 것입니다. 누군가는 컴퓨터와 대화를 하는 과정이라고도 할 것이고, 컴퓨터에게 명령을 내리는 것이라고도 생각할 수도 있습니다. 다 맞는 말입니다. 어떻게 생각하든 상관 없습니다. 정답은 없으니까요. 중요한 것은 프로그래밍이라는 것은 결국 인간이 하는 일이고, 인간이 내린 명령을 컴퓨터가 수행하게끔 한다는 것에 있습니다.

프로그래밍은 말 그대로 프로그램을 만드는 행위입니다. 여기서 프로그램이라는 것은 인간이 반복적으로 수행하는데에 있어서 번거롭거나 난해한 작업을 컴퓨터에게 수행하게끔 만든 소프트웨어라고 할 수 있습니다.

우리가 사용하고 있는 컴퓨터의 OS(Operating System)도 사실은 프로그램입니다. OS는 윈도우나 맥같은 운영체제를 부르는 단어입니다. 우리가 컴퓨터를 켜자마자 보이지 않는 영역에서 수많은 코드가 실행되고 돌아가고 있는 셈입니다.

그렇기 때문에 프로그래밍은 컴퓨터에게 명령을 내리는 과정이라고도 이해할 수 있겠습니다. 또한 프로그램을 만든다라는 표현도 컴퓨터에게 명령을 내린다라는 의미로 볼 수 있겠죠. 그렇다면 명령은 어떻게 내리는 것일까? 이 부분에서 프로그래밍 언어라는 개념이 처음으로 등장합니다. 프로그래밍 언어를 다루고 컴퓨터에게 명령을 내리는 사람을 우리는 프로그래머라고 부릅니다.

결론적으로 우리는 프로그래머가 되려는 것입니다.

# 프로그래밍 언어?
프로그램을 만들기 위해서 우리는 컴퓨터에게 명령을 내려야 했습니다. 명령을 내리기 위해서 컴퓨터가 이해할 수 있는 언어같은 것이 필요했고 그것이 바로 프로그래밍 언어입니다. 결론적으로 컴퓨터에게는 프로그래밍 언어를 통해서 명령을 내립니다. 프로그래밍 언어는 컴퓨터도 이해할 수 있고 인간도 이해할 수 있는 컴퓨터와 인간 사이에 약속된 언어를 말합니다. 프로그래밍 언어는 아주 다양합니다. 우리가 앞으로 배울 **자바스크립트(JavaScript)**도 프로그래밍 언어의 일종입니다.

* 자바(JAVA)
* 파이썬(Python)
* C언어 계통 (C+ / C++ / C#)
* 루비(Ruby)
* Go
* PHP

등등 세상에는 위에 언급한 것 말고도 아주 다양한 프로그래밍 언어가 존재합니다. 이 각각의 언어들은 사용처가 다를 뿐이지 전부 프로그래밍 언어인 것입니다. 각각의 문법이 있고, 그렇기 때문에 모양이 제각각 다르죠. 들어본 언어도 있을 것이고 그렇지 못한 것도 있을 겁니다. 걱정하지 마세요. 지금은 저런 언어들을 몰라도 됩니다. 우리가 다룰 자바스크립트라는 프로그래밍 언어만을 일단 기억해두면 되겠습니다.

자, 그럼 이제는 웹에 대해서 생각해볼 시간입니다. 프로그래밍 이야기를 먼저 한 것은 웹 또한 프로그램의 일종이라는 것을 다루기 위함이었습니다. 중요한 것은 이미 바로 위에서 언급되었습니다. 웹도 프로그램이라는 것입니다. 이 이야기를 하려면 세 가지 정도의 개념에 대해서 알아야 할 것 같습니다. 함께 알아보도록 합시다.

# 웹?
위에서 웹이라는 단어를 꺼내긴 했지만 웹에 대해 생소한 분들도 있을 것입니다. 웹은 일반적으로 구글이나 네이버등등의 사이트들이 존재하는 커다란 네트워크의 공간을 말합니다. 우리가 브라우저를 통해서 보고 있는 수많은 사이트들을 바로 웹이라고 부르는 것입니다.

# 인터넷과 웹의 관계
현재 우리가 사용하고 있는 인터넷의 근간에 대해 알고 있는 사람은 별로 없을 겁니다. 인터넷을 처음 고안한 사람은 **팀 버너스 리**라는 사람입니다. **인터넷의 아버지**라고 불리는 이 사람은 처음으로 인터넷을 고안했고, 인터넷의 존재로 인해 세계는 빠르게 가까워질 수 있었습니다.

**WWW**라는 단어에 대해서 들어봤을 것입니다. 이 WWW가 **월드 와이드 웹(World Wide Web)**의 약자입니다. 인터넷의 수많은 페이지들이 거미줄처럼 얽혀있다고 해서 웹(Web)이라는 이름이 붙었습니다 웹이 거미줄처럼 얽혀있는 이유는 하이퍼 링크<NoteTop number={'1'}>1</NoteTop>가 있기 때문입니다. 웹의 어디를 가나 링크가 있다는 것 정도는 이해할 수 있을 겁니다. 여러분이 이 포스트로 접근할 수 있었던 이유도 바로 링크 덕분이죠.

여기서 알고 넘어가야 할 것은 인터넷과 웹은 다르다는 것입니다. 인터넷의 안에 웹이라는 시스템이 존재하는 것이지 웹이 인터넷 그 자체인 것은 아입니다. 사과는 과일이지만 과일은 사과가 아닌 것과 같다고 생각하시면 되겠습니다. 웹은 인터넷 안에 존재하는 하나의 시스템인 것입니다. 다른 사람들은 몰라도 웹을 다루는 우리는 이것을 구분할 수 있어야 합니다.

아무튼 위의 이미지들은 우리가 평소에 아주 많이 볼 법한 웹 사이트들입니다. 네이버와 유튜브죠. 우리가 크롬이나 익스플로러 같은 브라우저를 이용해 볼 수 있는 사이트들이 바로 웹이라는 것입니다.

## 웹 페이지와 웹 사이트
이제 세가지의 개념에 대해서 알아보도록 합시다.

우선 웹 페이지는 수많은 웹 하나 하나의 페이지를 말합니다. 쉽게 말해 네이버의 메인 페이지와 이메일 관리 페이지를 비롯한 하나 하나의 페이지가 바로 웹 페이지입니다. 이 페이지들이 하이퍼 링크로 연결되어 있는 것이죠.

웹 사이트는 서로 관련된 페이지들로 구성되어 있는 것입니다. 네이버 메인 페이지와 이메일 관리 페이지, 지식인 페이지 등등이 모여서 네이버라는 웹 사이트 하나를 이룹니다. 그렇다면 구글이나 유튜브나 모두가 웹 사이트라는 것을 이해할 수 있겠습니다.

## 웹 프로그램
![우리가 흔히 보는 유튜브의 모습. 유튜브는 웹 애플리케이션이라고 할 수 있다.](https://i.postimg.cc/gcvxP23L/K-20210401-170120.png)

그럼 이제 마지막입니다. 웹 프로그램은 무엇일까요? 웹이 처음 등장한 이래 웹 페이지들에는 **프로그래밍적인 측면이 하나도 없었습니다.** 그야말로 웹 페이지들을 묶어 놓은 문서 덩어리, 문서의 집합이었죠. 오늘날의 인터랙티브한 웹과 사용자간의 상호작용이 없었다는 것입니다.

그렇기 때문에 예전에는 웹 사이트를 웹 문서라고 부르기도 했습니다.  하지만 웹이 발전하면서 **HTML**에서 **CSS**가 분리되고 골격과 디자인을 보다 구체적으로 신경쓸 수 있게 되었고, 더 나아가 **자바스크립트**가 개발되면서 웹에서도 **프로그램적인 접근**이 가능해졌습니다.

오늘날 웹은 웹 사이트이면서 동시에 **웹 프로그램** 혹은 **웹 애플리케이션**이라고 부를만해진 것입니다. 그러나 모든 웹은 웹 사이트라고 할 지라도 모든 웹 사이트가 웹 애플리케이션인 것은 아닙니다.

웹의 세계는 계속해서 변하고 있습니다. 위의 개념도 확실하게 구분된 것은 아닙니다. 앞으로 좀 더 다양한 웹 기술들이 나타날 것이고, 점점 그 경계가 깨지고 있다는 것은 확실한 것이죠.

# 웹 프로그래밍
<P>
  결국 웹 프로그래밍이라는 것은 단순히 <Strong>웹 퍼블리싱</Strong><NoteTop number={'2'}>2</NoteTop>이나 <Strong>웹 디자인</Strong><NoteTop number={'3'}>3</NoteTop>을 넘어서 웹과 사용자의 상호작용이 가능한 인터랙티브한 프로그램을 구현하는 것이라고 할 수 있는 것입니다.
</P>

웹 프로그래밍은 크게 세가지의 부류로 나뉩니다.

하나는 **프론트 엔드(Front-End)**입니다. 화면에 보이는 부분을 프로그래밍하는 부류입니다. 단순히 레이아웃을 만들고 디자인을 입혀 결과물을 내는 것과는 다릅니다. 프로그램적인 기능을 구현하는 것에 초점이 맞춰져 있습니다.

또 하나는 **백 엔드(Back-End)**입니다. 프론트 엔드가 보이는 화면을 개발하는 것이라면, 백 엔드는 보이지 않는 부분을 개발합니다. 보통 데이터베이스, API를 개발하죠. 즉, 서버와 관련된 부류라고 보면 되겠습니다.

마지막 부류가 프론트 엔드와 백 엔드를 둘 다 할 줄 아는 **풀스택(Full Stak)**이다. 하나만 할 줄 알아도 직업으로서 갈 길이 생기는데, 두개를 다 할 수 있다면 더욱 길이 열리게 됩니다.

이렇게 말을 해도 초심자인 우리는 잘 이해가 가지 않습니다. 어쩔 수 없는 것입니다. 우리가 이제 곧 웹을 만들고, 발전시키는 과정을 스스로 체험할 것이지만 프로그래밍의 영역에 발을 들이는 것은 자바스크립트를 만나고 난 다음부터일 것이고 그 뒤에 알아도 이상하지 않습니다. 그렇기 때문에 이해가 가지 않는다고 의기소침하거나 의욕을 잃을 필요는 전혀 없다는 겁니다. 처음인데 모르는 것은 **당연한 것**이기 때문입니다.

웹 프로그래밍은 웹 개발이라고도 합니다. 그렇기 때문에 웹을 전문적으로 프로그래밍 하는 사람들은 웹 개발자라고 불리는 것입니다.

# 웹 프로그래밍에 쓰이는 언어
지금까지는 이 부분을 설명하기 위한 빌드업이었습니다. 웹 프로그래밍에 쓰이는 언어는 기본적으로는 총 세가지입니다.

웹의 뼈대이자 마크업을 담당하는 **HTML**. 웹의 피부이자 디자인, 스타일을 담당하는 **CSS**. 웹의 근육, 뇌이자 웹에 대한 전반적인 것들 컨트롤 하는 **자바스크립트** 정도가 있겠습니다.

사실 이 언어들 말고도 웹 프로그래밍에 쓰이는 언어는 다양한 편입니다. 왜 그런가하면 웹의 시장이 커지면서 다른 언어들도 웹 개발을 지원하는 경우도 생기고 있기 때문입니다. 지금 알 필요는 없으니 나중에 천천히 알아봐도 좋은 부분입니다.

아무튼간에 우리는 앞으로 이 세가지에 대해 배워 갈 것입니다. 함께 말이죠. 갈 길이 멀지도 모릅니다. 하지만 천천히 앞으로 나아가면 생각보다 빠르게 목표지점에 도달할 수 있을지도 모릅니다.
